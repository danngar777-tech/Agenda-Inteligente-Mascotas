# ADR-001: Arquitectura Cliente-Servidor

## Estado

Aceptado

## Contexto

El sistema requiere una estructura que permita separar la interfaz de usuario, la lógica de negocio y el almacenamiento de datos.

## Decisión

Se utilizará una arquitectura cliente-servidor para el desarrollo del Sistema de Agenda Inteligente para Mascotas.

## Justificación

Esta arquitectura facilita la organización de componentes, mejora el mantenimiento del sistema y permite futuras ampliaciones.

## Consecuencias

- Mejor separación de responsabilidades.
- Mayor facilidad de mantenimiento.
- Posibilidad de escalar el sistema en el futuro.
